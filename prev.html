Cat Collector

Last semester, I took the class CSCI 4448: Object-Oriented Analysis and Design. I worked with my friend to create a Cat Collector game, using the principles of Object-Oriented Programming.

The game was originally based on the mobile game Neko Atsume: Kitty Collector, in which players are able to use different treats and toys to lure various cat characters to their cat garden. My friend and I were both inspired by this game because we both have cats of our own.

Technical Information

The game is coded in Java with various Object-Oriented patterns (such as Factory, Strategy, etc.)

The player's game information is stored in a dynamic JSON file, stored with the rest of the game files. This allows a player's stats to be saved between games (such as number of cats attracted, inventory, etc.)

GitHub and Notion were used for project management and source code management.

Design Process

We started by designing an initial diagram of the app's infrastructure. We defined the client layer as the user's input (keyboard/mouse/etc.), the executable/code layer as the Java code we had written (which generated the GUI), and the data layer as the various text files we used to store game information.

From there, we began developing different features of the game, from designing the characters and sprites to defining the behavior and interactions between various game objects.

Finally, we began to incorporate object-oriented patterns into our game design. This prepared us to start coding the game!
